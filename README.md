# 有点晚拌川 · 出摊小程序

真正的微信小程序项目（不是网页），可以直接用「微信开发者工具」打开运行。

## 怎么打开

1. 下载并解压这个项目
2. 打开「微信开发者工具」
3. 选择「导入项目」，目录选择解压后的 `youdianwan` 文件夹
4. AppID 可以先选择「测试号」或者填入你朋友自己申请的小程序 AppID
5. 点击「编译」，就能看到效果

## 功能说明

**顾客端（pages/index）**
- 出摊状态（营业中 / 今日休息），带印章样式和呼吸灯
- 今日出摊地点、时间
- 拌川 / 饮品 分类切换菜单，每项带图标和价格

**老板端（pages/admin）**
- 页面右上角点「⚙ 老板设置」进入（真实上线前建议加个简单密码或只给老板自己看到入口）
- 一键切换营业中 / 今日休息
- 常用地点、常用时间段预设，点一下就选中
- 今日留言输入框
- 点「保存并发布给顾客」后，顾客端会立刻同步显示最新信息

## 现在数据存在哪里

现在用的是 `wx.setStorageSync` / `wx.getStorageSync`，也就是**本地缓存**：
- 优点：不用申请服务器和数据库，直接能跑
- 限制：数据存在"打开小程序的这台设备"本地。如果老板在自己手机上设置，顾客在自己手机上打开，是**看不到老板设置的最新数据的**，因为数据没有真正同步到云端

### 如果要让顾客真的能看到老板发布的最新出摊信息，需要下一步：

用微信官方的「云开发」（免费额度够这种小场景用）：
1. 在微信开发者工具里开通「云开发」
2. 建一个数据库集合，比如 `stallInfo`
3. 老板保存时改成调用 `wx.cloud.database().collection('stallInfo').doc('today').set({...})`
4. 顾客端 `onShow` 时改成 `wx.cloud.database().collection('stallInfo').doc('today').get()` 读取

这一步我也可以帮你接着做，把本地缓存换成云开发数据库，实现真正的"老板发布、所有顾客都能看到"。

## 替换成真实菜品照片

现在菜单缩略图默认是大 emoji（🍜🥩🍖🍳🍺🥤💧）占满图片区域。**现在图片区域已经改成了整卡片顶部的大图（300rpx 高），直接支持真实照片**，改法很简单：

1. 把拍好的照片放进 `miniprogram/images/` 目录，比如 `banchuan-1.jpg`（建议裁成接近 16:10 或 4:3 的横图，效果最好）
2. 打开 `pages/index/index.js`，给对应菜品加一个 `image` 字段，比如：
   ```js
   { name: '杭州老式拌川', hot: true, desc: '...', price: '¥15', image: '/images/banchuan-1.jpg' }
   ```
3. 不用改 wxml 和 wxss —— 只要数据里有 `image` 字段，就会自动显示真实照片；没有 `image` 字段的菜品会继续显示 emoji 占位图，方便你一道一道慢慢换照片，不用一次性全换完。

## 字体说明

之前网页版用了 Google Fonts（衬线体+手写体），微信小程序不能直接连 Google Fonts。现在用的是系统默认字体加粗体现层次。如果想要更有个性的字体，需要把字体文件（.ttf）放进项目里，用 `@font-face` 引入，但会增加小程序包体积，普通出摊小程序不建议这么做。

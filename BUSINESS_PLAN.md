# shoplet - 商家小程序平台业务规划

## 一、业务理念

### 1.1 使命
> **"让每一个小商家都能拥有自己的小程序"**

我们致力于为小商家提供简单、免费、专业的小程序解决方案，帮助他们摆脱平台佣金的束缚，建立自己的私域流量，服务好每一位回头客。

### 1.2 愿景
成为中国最大的小商家小程序服务平台，服务100万+小商家，让他们的生意更简单、更高效、更赚钱。

### 1.3 核心价值观
- **免费为本**：基础功能永远免费，只为增值服务收费
- **简单至上**：极简操作，商家几分钟就能上手
- **商家自主**：商家拥有自己的小程序，数据归自己所有
- **持续迭代**：倾听商家反馈，不断优化产品

## 二、业务范围

### 2.1 服务对象
- **摆摊商家**：路边摊、夜市、流动摊贩
- **小区周边水果店**：社区水果店、水果摊
- **服装店**：精品服装店、潮牌店、工作室
- **餐馆**：小餐馆、特色餐厅、小吃店

### 2.2 核心功能

#### 摆摊模板
- 实时营业状态（未出摊/营业中/已收摊）
- 实时位置定位 + 一键导航
- 出摊时间预告
- 简易菜单展示
- 公告通知

#### 水果店模板
- 商品分类展示（水果/果切/礼盒）
- 库存状态显示
- 距离与配送范围提示
- 今日特价

#### 服装店模板
- 多图商品展示
- 尺码选择器
- 上新动态
- 搭配推荐

#### 餐馆模板
- 菜单分类（凉菜/热菜/主食/酒水）
- 桌号选择
- 营业时间提示
- 招牌菜推荐

### 2.3 平台功能
- 商家注册与认证
- 模板选择与配置
- 一键发布小程序
- 皮肤商店
- 数据统计与分析

## 三、业务逻辑

### 3.1 商家入驻流程
```
商家搜索我们的平台小程序
    │
    ├── 微信一键登录
    │
    ├── 填写商家信息
    │   ├── 小程序AppID
    │   ├── 代码上传密钥
    │   └── 联系方式
    │
    ├── 选择模板类型
    │   ├── 🍜 摆摊模板
    │   ├── 🍎 水果店模板
    │   ├── 👔 服装店模板
    │   └── 🍳 餐馆模板
    │
    ├── 配置店铺信息
    │   ├── 店铺名称
    │   ├── 联系方式
    │   ├── 店铺图片
    │   └── 商品信息
    │
    └── 点击发布
        ├── 后端生成商家专属代码包
        ├── 使用miniprogram-ci上传到商家小程序
        └── 提示商家在微信公众平台提交审核
```

### 3.2 模板更新流程
```
我们更新模板代码
    │
    ├── 后台发布新版本
    │
    ├── 通知商家有新版本可用
    │
    ├── 商家在平台小程序点击"更新"
    │
    └── 自动重新生成并上传代码包
```

### 3.3 皮肤购买流程
```
商家进入皮肤商店
    │
    ├── 浏览皮肤列表
    │
    ├── 预览皮肤效果
    │
    ├── 选择皮肤 → 微信支付
    │
    ├── 购买成功 → 自动应用到商家小程序
    │
    └── 重新发布小程序 → 皮肤生效
```

### 3.4 用户访问流程
```
用户扫码/搜索商家小程序
    │
    ├── 查看店铺状态
    │
    ├── 浏览商品列表
    │
    ├── 一键导航到店铺
    │
    └── 联系商家
```

### 4.2 核心技术组件

#### 后端服务
- **框架**: Express.js
- **数据库**: MySQL + Redis
- **小程序上传**: miniprogram-ci
- **支付**: 微信支付API
- **服务器**: Node.js

#### 小程序开发
- **框架**: 微信小程序原生框架
- **样式**: WXSS + CSS变量
- **状态管理**: 小程序数据绑定
- **云开发**: 可选（商家自己配置）

### 4.3 代码包生成流程

```
模板代码（通用）
    │
    ├── 替换 AppID
    ├── 替换商家配置（从数据库读取）
    ├── 应用皮肤样式
    ├── 生成 project.config.json
    ├── 生成 app.js 配置
    └── 生成商家专属代码包
```

### 4.4 批量更新机制

```
我们更新模板代码
    │
    ├── 后台发布新版本
    │
    ├── 选择需要更新的商家
    │
    ├── 逐个生成商家专属代码包
    │
    ├── 使用miniprogram-ci批量上传
    │
    └── 记录上传状态并通知商家
```

### 5.7 竞争策略

#### 差异化优势
1. **零佣金**：不收取交易流水佣金，相比美团18%-25%的佣金，每年为商家节省数万元
2. **完全免费**：基础功能永久免费，只有微信官方收取的300元认证费
3. **私域流量**：商家拥有自己的小程序，用户数据归自己，不受平台算法控制
4. **一键发布**：极简操作，几分钟拥有小程序，不需要懂技术
5. **皮肤系统**：其他平台没有的个性化定制服务

#### 市场定位
> **"做小商家的第一选择，而不是唯一选择"**

我们专注于小商家，提供简单、免费、专业的服务。不与美团、饿了么等大平台竞争，而是做他们的补充——帮助商家建立自己的私域流量，服务好回头客。

## 七、风险评估

### 7.1 技术风险
- **小程序更新**：微信小程序规则更新可能影响兼容性
- **服务器稳定性**：高并发时可能出现性能问题
- **安全风险**：商家密钥和用户数据的安全

### 7.2 市场风险
- **竞争加剧**：其他平台可能推出类似服务
- **商家流失**：商家可能转向其他平台
- **需求变化**：商家需求可能发生变化

### 7.3 运营风险
- **审核风险**：小程序审核可能被驳回
- **支付风险**：微信支付可能出现问题
- **客服压力**：商家咨询量可能过大

### 7.4 应对策略
- **技术风险**：定期更新代码，使用云服务器自动扩容，加密存储敏感数据
- **市场风险**：持续创新，提供差异化服务，建立商家社区
- **运营风险**：建立审核流程，备用支付渠道，自动化客服系统
